# 从日记生成周记

## 目标

根据一周内的日记记录、周记模板以及补充证据，生成一份简洁、可追溯、尽量不编造的周记草稿。

## 使用时机

适用于以下场景：

- 用户希望把一周的日记整理成周记或周复盘
- 已经有固定周记模板，需要根据每天记录自动填充
- 需要在周记中保留结论与原始记录之间的证据关联
- 需要把健身、睡眠、任务完成情况等分散信息汇总成周视角

不适用于以下场景：

- 没有日记原文，只要求凭空生成周记
- 用户想写的是月报、季度总结或纯创意文案
- 需要输出与原模板完全无关的自由格式长文

## 输入

必需输入：

- 一周范围内的日记记录
- 周记模板，或明确的目标输出结构

可选输入：

- 健身记录
- 睡眠记录
- 任务完成记录
- 指定周范围

默认处理规则：

- 如果用户未指定周范围，默认按周一到周日理解
- 如果当前周数据尚未结束，应明确标记“本周数据未闭合”
- 如果缺少部分日记，只整理已有证据，不补写未知内容
- 如果没有单独健身记录，可从日记中回退提取相关信息

## 工作流程

1. 确认范围。
   明确本次周记对应的日期范围，优先使用用户指定区间，否则按周一到周日处理。

2. 收集原始材料。
   读取该周日记、周记模板，以及可选的健身或其他辅助记录。

3. 只提取有证据的信息。
   优先提取以下内容：
   - 本周重点事项
   - 任务完成情况
   - 反思与感受
   - 学习进展
   - 睡眠与精力状态
   - 健身频率与训练内容
   - 一句话周总结

4. 轻量计算指标。
   在可计算时，整理以下指标：
   - 睡眠时长或平均睡眠
   - 重点任务完成次数
   - 健身次数

5. 填充模板。
   保持原周记模板的大体结构，用简洁、可审查的表达填写内容。

6. 保留可追溯性。
   对关键判断、关键数据、关键结论附上来源引用或来源标识，避免无法追溯。

7. 处理不确定项。
   对证据不足的字段留空，或明确标注“信息不足”，不要为了完整性编造内容。

## 输出格式

输出应为一份结构化周记草稿，通常包含：

- 本周概览
- 关键进展
- 任务完成情况
- 学习与成长
- 健康或健身情况
- 问题与反思
- 下周重点
- 证据引用

输出要求：

- 语气简洁、克制、可复查
- 模板结构优先，其次才是文风润色
- 关键结论应尽量带来源
- 不确定内容必须显式保守处理

## 示例

示例请求：

“把这周的日记整理成周记，使用周复盘模板，并附上关键结论对应的原始记录链接。”

示例结果特征：

- 保留周记模板的标题和分区
- 从每日记录中提取任务推进、学习内容、情绪状态
- 对“本周共锻炼 3 次”这类结论附上对应来源
- 如果周日尚未写完，明确备注“本周尚未结束”

## 边界情况

- 如果某天日记缺失，不推测该天发生了什么
- 如果不同来源对同一事实有冲突，优先使用更直接、更结构化的记录
- 如果时间字段是四位数字，例如 `0053`、`0800`，可按 `HH:MM` 解析
- 如果模板和原始记录明显不匹配，应先保留模板结构，再尽量只填充有证据的部分
- 如果健身信息在专门记录和日记里不一致，优先信任专门健身记录

## 安全与隐私

- 不在缺乏证据时补写用户状态、情绪或结论
- 不夸大完成度、效率或成长结果
- 若周记涉及敏感个人信息，输出时应优先最小化暴露
- 保持“基于记录整理”，而不是“替用户虚构更好看的总结”
